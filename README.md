# Ira — Cross-App Intelligence

React Native app with Android and iOS home-screen widgets, native context bridges via Expo Modules, permission-aware onboarding, and a rule-based suggestion engine.

## Architecture

Uses **Continuous Native Generation (CNG)** — the `android/` and `ios/` folders are generated by `npx expo prebuild` and not committed to git. All native code lives in `modules/expo-context-signals/`.

```
modules/expo-context-signals/
├── android/          Expo Module (Kotlin) + widget + notification listener
├── ios/              Expo Module (Swift)
├── ios-widget-extension/  WidgetKit files (manual Xcode setup required)
├── src/              TypeScript module binding
└── index.ts
```

## What is implemented

- Android widget (RemoteViews) with small/medium/large variants
- iOS WidgetKit extension with small/medium/large/lock screen support
- Expo Modules native bridge (calendar, contacts, music, usage, messages on Android; calendar, contacts, music on iOS)
- Health Connect integration via `react-native-health-connect`
- Rule-based contextual suggestion engine
- Progressive one-at-a-time permission onboarding
- Shared widget payload generation and sync
- Snapshot normalization with on-device caching and TTL
- Android periodic widget refresh via WorkManager-compatible update cycle
- iOS widget timeline refresh with live EventKit/HealthKit/MediaPlayer data fetching
- Android notification-based message summaries
- Pull-to-refresh on home screen
- Error boundary

## Commands

```bash
pnpm start              # Start Metro
pnpm android            # Build and run Android
pnpm ios                # Build and run iOS
pnpm test               # Run Jest tests
pnpm exec tsc --noEmit  # Type check
```

### First-time setup

```bash
pnpm install
npx expo prebuild
# Create android/local.properties with sdk.dir=<path>
pnpm android
```

### After clean prebuild

The iOS widget extension requires manual Xcode setup — see `modules/expo-context-signals/ios-widget-extension/README.md`.

## Project structure

- `App.tsx` — app shell with tab navigation
- `src/screens/` — Home, Sources, Onboarding
- `src/modules/intelligence/` — types, hooks, engine, bridge, tests
- `src/components/` — PageLayout, ErrorBoundary
- `modules/expo-context-signals/` — all native code (Expo Module)
- `plugins/withIraNativeConfig.js` — config plugin for manifest entries
- `docs/` — architecture, widgets, permissions, testing, tradeoffs
